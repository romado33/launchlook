What LaunchLook Verified means

Short version: a real person walked through the site and ran our usual checks. Long version below. Read it before drawing strong conclusions either way.

What Verified means

We ran our checkup on this site within the last 30, 90, or 180 days (the exact window depends on the tier the operator paid for; see the badge page or the pricing list). No exposed credentials were visible in the HTML we read, broken CTAs were noted and flagged as present at audit time, and the mobile layout was passable at common breakpoints.

That checkup is a human pass over what a first-time visitor would see and touch. Findings include polish, trust pages, broken links, dead CTAs, leftover dev tools, layout issues at phone width, and a small set of visibility-only security checks (admin routes loading without a login, API keys visible in page source, that sort of thing).

What Verified does NOT mean

  • Not a security audit. We do not pen-test, fuzz, or test for OWASP top 10.
  • Not an accessibility statement. We run a baseline check, not a full conformance review.
  • Not a promise that future changes will not break things. Sites change after the badge is issued.
  • Not a code review. We never see the codebase.
  • Not a financial, legal, or medical sign-off for anything the operator does on the site.

If you need any of the above, the badge is the wrong primitive. Look for a real pen-test firm, a real accessibility specialist, or the relevant regulator in your country.

Why it's still useful

Site visitors can see the badge and know the operator cared enough to get a third-party once-over before going live. That reduces the "is this a scam?" friction new visitors feel when they land on an unfamiliar site, especially for vibe-coded apps where the polish-to-stakes ratio is high.

The badge is a small social signal, not a stamp of approval from an authority. We tell visitors exactly that, here and on every verification page.

Validity windows

The window matches the tier the operator paid for. After the window closes, the badge stops verifying as current and a re-check is needed.

  • Starter Package ($19): 30 days.
  • Scale Up Package ($49): 90 days.
  • Pro Package ($99): 180 days.

Re-verification is $9 for sites with an active or recently expired badge. The re-check runs the same playbook against the live site, notes what changed, and reissues a fresh badge for the same window.

Questions a thoughtful visitor might still ask

Can the operator forge a badge?+

They could embed an image that looks like ours, but the link target points to launchlook.app/verify?slug=.... Click the badge. If our verification page does not recognize the slug or shows it as expired, the badge is not current.

What if the site changed since the audit?+

The badge stays valid for the window listed above. Operators are encouraged to re-check after meaningful changes; the $9 re-verification exists for exactly that.

Why no fancy seal?+

Because LaunchLook is a one-person checkup, not a standards body. A fancy seal would over-promise. A small monochrome badge says what it says: a human took a look.
