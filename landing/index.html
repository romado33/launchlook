Process

How it works

  1. 1

    You send your live URL

    Through our 30-second intake form: app URL, builder (Lovable, Bolt, v0, Cursor, or similar), one-line description.

  2. 2

    We walk every screen

    Every screen on desktop and phone. We check the things a first-time visitor would notice: broken buttons, placeholder copy, missing legal pages, mobile layout breaks, obvious visible risks like exposed credentials or public admin routes.

  3. 3

    A founder personally reviews every finding

    Every report is reviewed by Rob (the founder) before it ships. Wording that doesn’t land gets cut, anything ungrounded in real evidence gets dropped, and only the things you should actually fix make it into the final PDF.

  4. 4

    You get two PDFs by email

    Your Main Report and Quick Start Guide arrive in your inbox. No login, no dashboard, no extra account. Prioritized findings, screenshots, fix prompts.

Scope

What we catch

  • 1 Embarrassing polish: dead CTAs, broken links, layout breaks on phone width, dev leftovers on your public URL.
  • 2 Leftover placeholders: "Your Company Name," lorem ipsum, default Lovable/Bolt copy, fake email addresses still showing.
  • 3 Trust gaps: missing privacy/terms, generic meta, anything that screams “not finished.”

On Scale Up ($49) and Pro ($99) we also check whether signed-in users can see each other's data (you provide two temporary test accounts). Test logins are used only during the review and deleted within 24 hours. This is a basic visibility check, not a full security audit.

Questions

FAQ

Which platforms do you support?+

Any vibe-coded web app with a public URL. We often review Lovable, Bolt, Base44, and Replit apps. Also v0, Cursor-built apps, and others. If your builder outputs a shareable link, we can usually review it.

Is this only for mobile apps?+

No. We review web apps built with any vibe-coding tool (Lovable, Bolt, v0, Cursor, etc.). “Phone check” means we load your URL at mobile width in the browser, not the App Store.

What do you need from me?+

Your public or staging URL, a one-line description, and your builder. For Scale Up ($49) and Pro ($99), two temporary test accounts so we can check user data isolation.

Do you need access to my code or GitHub?+

No. We only need your public URL (and for Scale Up and Pro, two temporary test accounts). We never ask for GitHub access, your codebase, your builder login, or any API keys. If a tool ever asks for those things just to spot broken UI, be wary.

Is this just an AI scanner?+

No. Every report is personally reviewed by a founder before delivery. We use software to capture every screen and surface common pre-launch issues, then we read every finding, drop the false positives, fix wording that doesn't land, and only ship what you should actually act on.

How is this different from a security scanner?+

We focus on what visitors see: broken UI, placeholders, trust pages. For deep security, use something like VAS. Use both if you can.

Lovable already has SEO review. Why this?+

SEO tools help Google. We help the friend you send the link to: dead buttons, dev tools on prod, missing legal pages, fake-looking reviews.

How are you different from PageLens AI?+

Both scan your live site and surface findings. We're built for vibe coders shipping with Lovable / Cursor / Bolt; PageLens is built for SEO-focused marketers. Full comparison →

Staging URLs OK?+

Yes if they're reachable on the internet (Lovable preview, Vercel staging, etc.). localhost won't work.

How do I know my data is safe?+

We only access your app through the public URL and any temporary test accounts you provide. We don't request code access, database access, or API keys. Credentials are submitted through the intake form only and deleted within 24 hours of delivery. See our privacy policy.
