Use this when your **Report Templates** parent has three child pages. Import each Markdown file as its own Notion page, then mark each page as a **Notion template** (Duplicate).

| Notion page name | Source file in repo | Price | Findings cap | Extra sections vs previous setup |
|------------------|---------------------|-------|--------------|----------------------------------|
| **Template — Starter Package** | `report-quick-checkup.md` | $19 | 10 | Main Report + Pre-Launch Checklist PDFs only |
| **Template — Scale Up Package** | `report-launch-pack.md` | $49 | 30 | + two-account check, Quick Start Guide (was "Full Package" / `report-launch-pack`) |
| **Template — Pro Package** | `report-pro-package.md` | $99 | 40 | + integrations table, Loom block, Handoff pointer |

Legacy filenames are kept so old links in docs still work:

- `report-quick-checkup.md` = **Starter** (updated; was $9 Quick Checkup)

- `report-polish.md` = optional follow-up tier — **not** part of the current ladder

## Import into Notion (per page)

1. Open the existing template page (or create **Template — Pro Package** if new).
2. **⋯ → Import → Markdown** and choose the matching `.md` file from `templates/notion/`.
3. Replace the old page content, or import into a new page and delete the stale one.
4. Turn finding blocks into **toggles** if you prefer (Notion import keeps `##` headings).
5. **⋯ → Turn into template** (or use Notion's template button) so duplication stays one click.
